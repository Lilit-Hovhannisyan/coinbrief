# coinbrief

> AI-free intelligence briefings for any cryptocurrency — powered entirely by the [CoinStats Crypto API](https://coinstats.app/api/).

Type a coin. Get an analyst-grade briefing in seconds — generated by a built-in signal engine, grounded in live market data. No external AI API needed.

## What it does

**coinbrief** generates real-time intelligence briefings for any cryptocurrency using a two-layer architecture:

1. **CoinStats API fetches live data** — price, volume, market cap, multi-timeframe changes, 7-day charts, news headlines, and Fear & Greed index from the [CoinStats Crypto API](https://coinstats.app/api/)
2. **Built-in signal engine computes the analysis** — a JavaScript-based engine calculates 7 market signals, then generates a structured English briefing using data-driven templates

Each briefing includes:
- **What's happening** — current price action with volume and trend context
- **Signal scan** — color-coded bullish / bearish / neutral chips
- **What to watch** — key support/resistance levels and volume triggers
- **News context** — headline summary from CoinStats news feed
- **Verdict** — a 0-100 composite score with plain-English assessment

### No external AI API required

The entire analysis is computed client-side. The server is a thin proxy for the [CoinStats API](https://coinstats.app/api/) — it adds no AI calls, no third-party dependencies, and costs nothing beyond CoinStats' free tier.

## Tech stack

- **Frontend:** Vanilla HTML / CSS / JS — no framework, fast and simple
- **Backend:** Node.js + Express — proxies [CoinStats API](https://coinstats.app/api/) calls to protect your key
- **Data source:** [CoinStats API](https://coinstats.app/api/) — real-time prices, charts, news, and Fear & Greed for 100,000+ coins
- **Intelligence:** Built-in signal engine + briefing generator (pure JavaScript)

## Quick start

### Prerequisites

- [Node.js](https://nodejs.org/) 18+
- A free [CoinStats API key](https://openapi.coinstats.app) (no credit card needed)

### Setup

```bash
# Clone the repo
git clone https://github.com/YOUR_USERNAME/coinbrief.git
cd coinbrief

# Install dependencies
npm install

# Create your .env file
cp .env.example .env
# Edit .env and add your CoinStats API key

# Start the server
npm start
```

Open [http://localhost:3000](http://localhost:3000) and search for any coin.

## How the signal engine works

The engine computes 7 signals from raw [CoinStats API](https://coinstats.app/api/) data, then generates the briefing from those signals:

| Signal | What it measures | How it's computed |
|--------|-----------------|-------------------|
| **Momentum** | Overall direction (0-100) | Weighted average: 1h×0.15 + 24h×0.35 + 7d×0.30 + 30d×0.20 |
| **Volume ratio** | Trading intensity | 24h volume ÷ market cap × 100 |
| **Volatility** | Price swing magnitude | Max timeframe change − min timeframe change |
| **Trend alignment** | Multi-timeframe agreement | Count of positive vs negative timeframes |
| **Divergence** | Short vs long-term conflict | (1h + 24h) avg − (7d + 30d) avg |
| **Market tier** | Size classification | Rank-based: blue-chip / large / mid / small / micro |
| **Risk score** | Token safety | CoinStats native risk assessment (when available) |

The briefing generator uses conditional logic to write each section — choosing templates, combining sentences, and adjusting tone based on the computed signals. No AI API is called.

## CoinStats API endpoints used

All data comes from the [CoinStats Crypto API](https://coinstats.app/api/):

| Endpoint | Purpose | Docs |
|----------|---------|------|
| `GET /coins` | Search and list coins | [API docs](https://coinstats.app/api-docs/openapi/get-coins/) |
| `GET /coins/:id` | Detailed coin data with all timeframe changes | [API docs](https://coinstats.app/api-docs/openapi/get-coin-by-id/) |
| `GET /coins/:id/charts` | 7-day price chart for sparkline + trend analysis | [API docs](https://coinstats.app/api-docs/openapi/get-coin-chart-by-id/) |
| `GET /news` | Latest crypto news headlines | [API docs](https://coinstats.app/api-docs/openapi/get-news/) |
| `GET /markets/fear-greed` | Market Fear & Greed index | [API docs](https://coinstats.app/api-docs/openapi/fear-and-greed/) |
